## 환경 및 목적

Stack: React (또는 Next.js), Supabase Auth 사용
목표: custom:naver 프로바이더를 통해 로그인을 구현하고, 로그인 성공 시 profiles 테이블에 사용자 정보를 자동으로 저장/업데이트하는 로직 구현.

## 인증 설정 (OAuth)

Provider ID: custom:naver를 사용합니다.
필요 Scopes: email, nickname, profile_image, gender
보안: .env 파일은 직접 수정하거나 읽지 말고, 코드 내에서 process.env.REACT_PUBLIC_SUPABASE_URL 등 환경 변수 이름만 사용하여 구조를 잡아주세요. (키값은 내가 직접 넣을 예정)

## 데이터 매핑 로직 (핵심)

네이버 로그인이 성공하면 session.user.user_metadata 내의 데이터를 분석하여 아래 규칙에 따라 profiles 테이블에 데이터를 upsert 하거나 insert 하세요.
별명 (Nickname): - 네이버에서 제공한 nickname이 있다면 최우선으로 사용.
nickname이 없다면 이메일 주소의 @ 앞부분을 추출하여 사용.
성별 (Gender): - 네이버에서 정보를 가져왔다면 해당 값을 사용.
정보가 없다면 기본값을 '비밀'로 설정.
프로필 사진 (Profile Image): - 네이버에서 준 이미지 URL을 가져오되, 나중에 Storage에 올리거나 별도 DB에서 처리할 수 있도록 변수화하여 profiles 테이블의 avatar_url 컬럼에 저장.
신규/기존 유저 판별:
profiles 테이블에 해당 user_id가 없다면 신규 가입으로 간주하고 초기 데이터를 생성.
이미 존재한다면 로그인으로 취급 (필요 시 최신 정보로 업데이트).

## 구현 요청 사항

onAuthStateChange를 사용하여 로그인 이벤트를 감지하는 로직을 작성해줘.
네이버는 응답 데이터가 response 객체 안에 한 번 더 감싸져 있을 수 있으니, 데이터 경로를 안전하게 파싱하는 로직(Optional Chaining 등)을 포함해줘.
profiles 테이블에 들어갈 nickname, gender, avatar_url, last_check_in_at 등의 필드를 포함한 insert 쿼리를 작성해줘.
